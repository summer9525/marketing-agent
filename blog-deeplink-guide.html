안녕하세요, 디파이너리입니다.

앱 마케팅을 하다 보면 "딥링크를 설정해주세요"라는 말을 자주 듣게 됩니다. 그런데 막상 딥링크를 파고들면 URI 스킴, 유니버셜 링크, 앱링크… 비슷해 보이는 용어들이 쏟아져 나옵니다.

이 글에서는 딥링크의 개념부터 세 가지 방식의 차이, 그리고 실무에서 어떻게 활용해야 하는지까지 한 번에 정리해드리겠습니다.

딥링크(Deeplink)란?

웹에서 https://igaworks.com을 입력하면 아이지에이웍스 홈페이지로 이동하듯, 모바일 앱에서도 특정 주소를 통해 앱을 실행하거나 앱 내 특정 화면으로 바로 이동시킬 수 있습니다. 이것이 딥링크입니다.

딥링크가 적용되면 광고를 클릭한 고객은 앱이 바로 열리면서 원하는 상품 페이지로 이동하게 됩니다. 앱이 설치되어 있지 않다면 마켓으로 이동해 설치 후 해당 페이지로 연결되는 '지연된 딥링크(Deferred Deeplink)'도 가능합니다.

즉, 딥링크는 광고 성과를 극대화하기 위해 반드시 활용해야 하는 기능입니다.

1. URI 스킴 방식 — 가장 기본적인 딥링크

어떻게 작동하나요?

URI 스킴은 앱마다 고유한 스킴(Scheme) 값을 등록해두고, 해당 값을 호출하면 앱이 열리는 방식입니다.

구조는 간단합니다:

Scheme://Path
Scheme = 앱을 특정 (예: twitter)
Path = 앱 내 화면을 특정 (예: signup)

트위터 앱을 열고 싶다면 twitter://, 트위터의 회원가입 화면으로 바로 가고 싶다면 twitter://signup을 호출하면 됩니다.

URI 스킴의 한계

앱 수가 적던 시절에는 문제가 없었지만, 앱이 폭발적으로 늘어나면서 스킴값이 중복되는 문제가 발생했습니다.

대표적인 예로, 구글플레이·원스토어·삼성 앱스토어가 모두 market://이라는 동일한 스킴값을 사용합니다. 이 경우 사용자에게 "어떤 앱으로 열까요?"라는 선택 팝업이 뜨게 되죠.

개발자 입장에서는 자신이 등록한 스킴값이 다른 앱과 중복되는지 확인할 방법도, 막을 방법도 없습니다. 이 문제를 근본적으로 해결하기 위해 등장한 것이 바로 유니버셜 링크와 앱링크입니다.

2. 유니버셜 링크 & 앱링크 — 도메인 기반 딥링크

핵심 아이디어

인터넷 도메인 주소는 전 세계에서 고유합니다. igaworks.com은 아이지에이웍스만, dfinery.io는 디파이너리만 사용할 수 있죠.

애플과 구글은 이 고유한 도메인 주소를 딥링크 실행값으로 사용하기로 했습니다. 스마트폰 브라우저에서 https://naver.com을 입력하면 네이버 앱이 바로 열리는 방식입니다.

  • iOS에서는 이를 유니버셜 링크(Universal Link)라고 부릅니다.
  • Android에서는 앱링크(App Link)라고 부릅니다.

도메인은 고유하기 때문에 URI 스킴처럼 중복 문제가 발생하지 않습니다.

그런데, 이것도 완벽하진 않습니다

유니버셜 링크와 앱링크가 모든 앱 환경에서 동작하는 것은 아닙니다. 디파이너리 팀에서 주요 환경별로 직접 테스트한 결과를 공유합니다.

3. 디파이너리 All-Link — 세 가지를 하나로

URI 스킴, 유니버셜 링크, 앱링크 중 어느 하나만으로는 모든 환경을 커버할 수 없습니다. 그래서 디파이너리는 세 가지 방식을 하나의 트래킹 링크로 통합한 All-Link 기능을 제공합니다.

All-Link 작동 원리

  1. 유니버셜 링크 / 앱링크가 지원되는 환경 → 우선 실행
  2. 지원되지 않는 환경 → 자동으로 URI 스킴 방식으로 전환
  3. 앱 미설치 상태 → 마켓 이동 후 지연된 딥링크로 특정 화면 연결

마케터는 딥링크 방식별 호환성을 일일이 신경 쓸 필요 없이, 디파이너리 트래킹 링크 하나만 발급하면 됩니다. 어떤 환경에서든 고객이 광고를 클릭하면 앱이 열리고, 의도한 화면으로 정확히 이동합니다.

실무 체크리스트: 딥링크 설정 전 확인할 것

딥링크는 디파이너리가 '문을 여는 역할'을 하지만, '열쇠'는 앱 자체에 먼저 설정되어 있어야 합니다.

마무리

딥링크는 앱 마케팅의 기본이자 핵심입니다. 정리하면:

  • URI 스킴 — 가장 기본적이지만 스킴값 중복 문제가 있음
  • 유니버셜 링크(iOS) / 앱링크(Android) — 도메인 기반으로 중복 문제 해결, 하지만 모든 앱에서 동작하진 않음
  • 결론 — 세 가지 방식을 모두 적용해 상호 보완적으로 사용하는 것이 최선

디파이너리의 All-Link는 이 세 가지를 하나의 트래킹 링크로 통합해, 어떤 환경에서든 끊김 없는 딥링크 경험을 제공합니다. 광고 UX 최적화와 정확한 어트리뷰션을 동시에 잡고 싶다면, 디파이너리 All-Link를 활용해보세요.
