# 요구사항 문서

## 소개

모바일인덱스 API를 연동하여 오디언스 비교/분석 기능을 IGAWorks 웹사이트에 추가하는 기능입니다. 사용자는 최대 5개의 앱을 선택하여 오디언스 그룹을 구성하고, 최대 2개의 그룹 간 주요 지표(MAU, 신규 설치자 수, 충성 고객 지표, 고착도, 교차 사용 지표, 신규 설치자 재방문율)를 비교·분석할 수 있습니다. 그룹 내 중복 사용자는 자동으로 제거되며, 분석 결과를 오디언스로 추출하여 트레이딩웍스 360에서 광고 캠페인에 활용할 수 있습니다.

## 용어 정의

- **오디언스_분석_페이지**: 오디언스 비교/분석 기능을 제공하는 웹 페이지 (React 컴포넌트)
- **앱_검색기**: 모바일인덱스 API를 통해 앱을 검색하고 선택할 수 있는 UI 컴포넌트
- **오디언스_그룹**: 사용자가 선택한 최대 5개의 앱으로 구성된 분석 단위. 그룹 내 앱 간 중복 사용자가 제거된 상태
- **지표_대시보드**: 선택된 오디언스 그룹의 주요 지표를 시각화하여 표시하는 UI 영역
- **모바일인덱스_API_클라이언트**: 모바일인덱스 API와 통신하는 서비스 모듈
- **오디언스_추출기**: 분석 결과를 마케팅 활용 가능한 오디언스 데이터로 변환하는 모듈
- **MAU**: 월간 활성 사용자 수 (Monthly Active Users)
- **고착도**: DAU를 MAU로 나눈 비율 (DAU/MAU × 100)로, 앱의 일일 사용 빈도를 나타내는 지표
- **교차_사용_지표**: 서로 다른 앱을 동시에 사용하는 사용자의 비율 및 수
- **트레이딩웍스_360**: IGAWorks의 광고 집행 플랫폼

## 요구사항

### 요구사항 1: 앱 검색 및 선택

**사용자 스토리:** 분석가로서, 모바일인덱스에 등록된 앱을 검색하고 선택하여 오디언스 그룹을 구성하고 싶다.

#### 인수 조건

1. WHEN 사용자가 앱 이름 또는 패키지명을 입력하면, THE 앱_검색기 SHALL 모바일인덱스_API_클라이언트를 통해 일치하는 앱 목록을 표시한다
2. WHEN 사용자가 검색 결과에서 앱을 선택하면, THE 앱_검색기 SHALL 해당 앱을 현재 오디언스 그룹에 추가한다
3. WHILE 오디언스 그룹에 5개의 앱이 포함된 상태에서, THE 앱_검색기 SHALL 추가 앱 선택을 비활성화하고 최대 선택 수 도달 안내 메시지를 표시한다
4. WHEN 사용자가 그룹 내 앱의 제거 버튼을 클릭하면, THE 앱_검색기 SHALL 해당 앱을 오디언스 그룹에서 제거한다
5. THE 앱_검색기 SHALL 각 앱의 아이콘, 앱 이름, 플랫폼(AOS/iOS) 정보를 검색 결과에 표시한다

### 요구사항 2: 오디언스 그룹 관리

**사용자 스토리:** 분석가로서, 최대 2개의 오디언스 그룹을 생성하고 관리하여 비교 분석의 기준을 설정하고 싶다.

#### 인수 조건

1. THE 오디언스_분석_페이지 SHALL 최대 2개의 오디언스 그룹 생성을 허용한다
2. WHEN 사용자가 새 그룹 추가 버튼을 클릭하면, THE 오디언스_분석_페이지 SHALL 빈 오디언스 그룹을 생성하고 앱 선택 UI를 표시한다
3. WHILE 2개의 오디언스 그룹이 존재하는 상태에서, THE 오디언스_분석_페이지 SHALL 새 그룹 추가 버튼을 비활성화한다
4. WHEN 사용자가 그룹 삭제 버튼을 클릭하면, THE 오디언스_분석_페이지 SHALL 해당 오디언스 그룹과 포함된 앱 선택을 모두 제거한다
5. THE 오디언스_분석_페이지 SHALL 각 오디언스 그룹에 사용자 지정 이름을 설정할 수 있는 입력 필드를 제공한다

### 요구사항 3: 중복 사용자 제거

**사용자 스토리:** 분석가로서, 오디언스 그룹 내 앱 간 중복 사용자가 자동으로 제거되어 정확한 순 사용자 수 기반의 분석을 수행하고 싶다.

#### 인수 조건

1. WHEN 오디언스 그룹에 2개 이상의 앱이 포함되면, THE 모바일인덱스_API_클라이언트 SHALL 그룹 내 앱 간 중복 사용자를 제거한 순 사용자 데이터를 요청한다
2. THE 지표_대시보드 SHALL 중복 제거 전 합산 사용자 수와 중복 제거 후 순 사용자 수를 함께 표시한다
3. IF 모바일인덱스 API가 중복 제거 데이터를 반환하지 못하면, THEN THE 지표_대시보드 SHALL "중복 제거 데이터를 불러올 수 없습니다" 오류 메시지를 표시한다

### 요구사항 4: 지표 대시보드

**사용자 스토리:** 분석가로서, 오디언스 그룹의 주요 모바일 지표를 한눈에 확인하여 앱 시장 동향을 파악하고 싶다.

#### 인수 조건

1. WHEN 오디언스 그룹에 1개 이상의 앱이 선택되면, THE 지표_대시보드 SHALL 다음 6개 지표를 표시한다: MAU, 신규 설치자 수, 충성 고객 지표, 고착도(DAU/MAU×100), 교차 사용률 및 교차 사용자 수, 신규 설치자 재방문율
2. THE 지표_대시보드 SHALL 각 지표를 숫자 값과 차트(라인 차트 또는 바 차트)로 시각화한다
3. THE 지표_대시보드 SHALL AOS와 iOS 통합 데이터를 기본으로 표시한다
4. WHEN 사용자가 조회 기간을 변경하면, THE 지표_대시보드 SHALL 선택된 기간에 해당하는 지표 데이터를 다시 조회하여 표시한다
5. IF 모바일인덱스 API 호출이 실패하면, THEN THE 지표_대시보드 SHALL 해당 지표 영역에 "데이터를 불러올 수 없습니다. 다시 시도해 주세요." 오류 메시지와 재시도 버튼을 표시한다

### 요구사항 5: 그룹 간 및 그룹-앱 간 비교

**사용자 스토리:** 분석가로서, 두 오디언스 그룹 간 또는 그룹과 개별 앱 간 지표를 비교하여 경쟁 분석을 수행하고 싶다.

#### 인수 조건

1. WHEN 2개의 오디언스 그룹이 존재하면, THE 지표_대시보드 SHALL 두 그룹의 동일 지표를 나란히 비교하는 뷰를 제공한다
2. WHEN 사용자가 비교 대상으로 그룹과 개별 앱을 선택하면, THE 지표_대시보드 SHALL 선택된 그룹과 앱의 지표를 나란히 비교하는 뷰를 제공한다
3. THE 지표_대시보드 SHALL 비교 뷰에서 각 지표의 차이값과 변화율(%)을 표시한다
4. THE 지표_대시보드 SHALL 비교 대상 간 지표 추이를 하나의 차트에 오버레이하여 표시한다

### 요구사항 6: 모바일인덱스 API 연동

**사용자 스토리:** 개발자로서, 모바일인덱스 API와 안정적으로 통신하여 정확한 데이터를 제공하고 싶다.

#### 인수 조건

1. THE 모바일인덱스_API_클라이언트 SHALL API 인증 키를 환경 변수(.env)에서 읽어 요청 헤더에 포함한다
2. THE 모바일인덱스_API_클라이언트 SHALL 모든 API 응답을 TypeScript 타입으로 정의하고 파싱한다
3. WHEN API 응답을 수신하면, THE 모바일인덱스_API_클라이언트 SHALL 응답 데이터를 내부 데이터 모델로 변환한다
4. FOR ALL 유효한 API 응답에 대해, 파싱 후 내부 모델로 변환한 뒤 다시 API 요청 형식으로 직렬화하면 원본 요청과 동등한 객체를 생성한다 (라운드트립 속성)
5. IF API 요청이 네트워크 오류로 실패하면, THEN THE 모바일인덱스_API_클라이언트 SHALL 최대 3회까지 지수 백오프 방식으로 재시도한다
6. IF 재시도 후에도 API 요청이 실패하면, THEN THE 모바일인덱스_API_클라이언트 SHALL 호출자에게 구체적인 오류 정보를 포함한 에러 객체를 반환한다

### 요구사항 7: 오디언스 추출 및 마케팅 활용

**사용자 스토리:** 마케터로서, 분석된 오디언스 데이터를 추출하여 트레이딩웍스 360에서 광고 캠페인에 즉시 활용하고 싶다.

#### 인수 조건

1. WHEN 사용자가 오디언스 추출 버튼을 클릭하면, THE 오디언스_추출기 SHALL 현재 분석 중인 오디언스 그룹의 사용자 세그먼트 데이터를 생성한다
2. WHEN 오디언스 데이터가 생성되면, THE 오디언스_추출기 SHALL 트레이딩웍스 360 광고 집행 페이지로 이동할 수 있는 연결 링크를 제공한다
3. THE 오디언스_추출기 SHALL 추출된 오디언스의 예상 도달 규모(사용자 수)를 표시한다
4. IF 오디언스 추출 과정에서 오류가 발생하면, THEN THE 오디언스_추출기 SHALL "오디언스 추출에 실패했습니다. 다시 시도해 주세요." 오류 메시지를 표시한다

### 요구사항 8: 페이지 라우팅 및 레이아웃

**사용자 스토리:** 사용자로서, IGAWorks 웹사이트 내에서 오디언스 분석 페이지에 자연스럽게 접근하고 싶다.

#### 인수 조건

1. THE 오디언스_분석_페이지 SHALL `/audience-analytics` 경로로 접근 가능하다
2. THE 오디언스_분석_페이지 SHALL 기존 IGAWorks 웹사이트의 Navbar와 Footer 레이아웃을 동일하게 사용한다
3. THE 오디언스_분석_페이지 SHALL 반응형 레이아웃을 제공하여 데스크톱과 모바일 환경 모두에서 사용 가능하다
4. WHEN 페이지 로딩 중일 때, THE 오디언스_분석_페이지 SHALL 로딩 스피너를 표시한다

### 요구사항 9: 데이터 캐싱 및 성능

**사용자 스토리:** 사용자로서, 반복적인 분석 작업 시 빠른 응답 속도를 경험하고 싶다.

#### 인수 조건

1. THE 모바일인덱스_API_클라이언트 SHALL TanStack Query를 사용하여 API 응답을 캐싱한다
2. WHEN 동일한 앱 조합과 기간에 대한 데이터를 재요청하면, THE 모바일인덱스_API_클라이언트 SHALL 캐시된 데이터를 우선 반환하고 백그라운드에서 갱신한다
3. THE 모바일인덱스_API_클라이언트 SHALL 캐시 유효 기간을 5분으로 설정한다
